## 第一步：一键部署核心“机甲库”（工具链安装）

> 工欲善其事，必先利其器。

我们整合了所有基础依赖，只需一个脚本即可完成 **PX4 飞控、ROS2 框架和 Gazebo 仿真器** 的部署。

1. **获取脚本**：确保你拥有 `CH1_install_px4_ros2_gz_toolchain.sh` 脚本文件。
2. **执行安装**：在终端中运行以下命令。（**请勿使用 `sudo` 执行**）
   ```bash
   chmod +x CH1_install_px4_ros2_gz_toolchain.sh
   ./CH1_install_px4_ros2_gz_toolchain.sh
   ```

   这个脚本将自动完成：
   - ✅ ROS2 Humble/Jazzy 完整桌面版安装  
   - ✅ PX4-Autopilot 最新版源码下载与依赖配置  
   - ✅ 关键通信组件 `Micro-XRCE-DDS-Agent` 的编译安装  
   - ✅ ROS2 工作空间创建，并编译 `px4_ros_com` 等核心功能包  

3. **环境生效**：安装完成后，务必**关闭当前终端并重新打开**，或执行：
   ```bash
   source ~/.bashrc
   ```
   使所有环境变量生效。

---

## 第二步：启动“指挥中心”与下载基础场景

有了机甲库，我们还需要**指挥中心（地面站）** 和**训练场雏形**。运行第二个脚本：

```bash
chmod +x CH1_startup.sh
./CH1_startup.sh
```

此脚本将：
- 📁 初始化 Gazebo 的模型和世界文件目录 (`~/.simulation-gazebo/`)
- 🛰️ 下载并安装最新的 **QGroundControl (QGC)** 地面站软件
- ▶️ 尝试启动一个基础的仿真环境（包含 PX4 SITL、通信代理和 QGC）

> 💡 **提示**：首次运行时，你可以让脚本完整执行以验证基础链路。之后，可以按 `Ctrl+C` 终止脚本启动的 PX4 和 Gazebo 进程，但请保留 QGC，后续步骤仍需使用。

---

## 第三步：架设“感官神经”（安装 GZ Bridge）

要让你的算法“感知”到仿真世界，必须在 **ROS2 和 Gazebo 之间建立数据桥梁**。

```bash
# 请根据你的 ROS2 版本替换 `humble`
sudo apt update
sudo apt install ros-humble-ros-gz-bridge
```

安装后，你便能通过此桥接，将 Gazebo 中相机、激光雷达等传感器的数据，实时转换为 ROS2 标准话题，供你的感知与决策节点使用。

---

## 第四步：部署“省赛专属战场”与“战机”

现在，让我们用官方提供的、高度拟真的模型替换基础场景。

1. **克隆省赛资源库**：
   ```bash
   git clone https://github.com/YongxueLiu/ZheJiangShengSai-9-Aerial-robot-.git
   ```

2. **查看资源结构**：这里包含了赛场、机关和多种传感器配置的无人机模型。
   ```bash
   cd ZheJiangShengSai-9-Aerial-robot- && tree -L 2
   ```

   **关键内容**：
   - `worlds/drone_competition_v1.sdf`：1:1 还原的省赛场地世界文件，包含所有障碍物、竞速门、任务区。
   - `models/`：内含 `x290_mono_cam_down_lidar_3d_depth` 等多款搭载不同传感器的无人机模型，以及 `door`（竞速门）等道具模型。

3. **部署资源**：将它们复制到 Gazebo 的搜索路径。
   ```bash
   cp -r models/* ~/.simulation-gazebo/models/
   cp worlds/drone_competition_v1.sdf ~/.simulation-gazebo/worlds/
   ```

---

## 第五步：点火，起飞！

万事俱备，启动完整的省赛仿真环境：

```bash
./start_px4_gz_env_x290_mono_cam_down_lidar_3d_depth.sh
```

这个脚本将自动打开多个终端，并顺序启动：

1. **Gazebo**：加载 `drone_competition_v1` 赛场  
2. **PX4 SITL**：加载搭载单目相机和下视 3D 激光雷达的竞赛无人机模型  
3. **ROS2-GZ Bridge**：建立传感器数据桥接  
4. **MicroXRCE-DDS Agent**：建立通信链路  
5. **QGroundControl**：作为地面站监控  

✅ 当所有窗口就绪，你将在 Gazebo 中看到无人机静置于赛场 **H 区（起飞区）**，QGC 显示连接正常。

> 🎉 恭喜！你的虚拟战队已部署完毕！
